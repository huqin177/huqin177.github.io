# 前言

本人在大学以前几乎没接触过电脑，也算是家门第一人了（苦笑），稀里糊涂选了个工科专业，好巧不巧，选了个网络工程，还是偏软件方面的学习，代码的学习实在太迷茫了，都说各种语言有共通之处，但是我觉得区别还是很大的，难道就没一种语言通用的嘛，好吧，我自己看来也是个离谱的想法，太过于简化了机器，也把编程的学习想得太简单了。还有就是啊，老师们都说在课堂上学不了什么，想让我们自己去学，但是课堂管的又很严，竟然还出现了安装手机袋的情况，我觉得学习靠自觉，也靠督促，高中被推着前进，现在本人现在刚大二开学第三周，那破数据结构一点没听懂，还有个python（两周就结课了，四节课想让我们学到什么呢）。大一的时候啊，我被网上的言语吓倒了感觉，比如网上各种各样计算机已经过时了，这个专业没有前景啦，出来找不到工作，毕业即失业啦，一度让我毕业后不想从事计算机相关专业，那我又能干什么呢，考公考编吗，并不像。在我看来，不好找工作，是某些找不到工作的人制造的焦虑，他们不仅让自己陷于泥潭，也想拉别人下去，足够优秀的人是不愁找不到好工作的，而且还会有人找上门，这种人是极少的吧哈哈哈，有点异想天开了，想着自己会是那样的人哈哈哈，像做梦后实现嘛有意思哈，反正就是有硬实力的人不怕找不到好工作，如果你看到这儿了，也请相信自己，迷茫的人有很多，我现在也是，但不希望一直都是这样。

二、.github干什么的？

1.豆包是这样说的

.github 文件夹的核心规则可以总结为：“按 GitHub 约定的「文件名 / 目录结构」存放「特定格式」的文件，才能触发对应的功能”。具体规则可分为 “位置规则” 和 “内容规则” 两类，掌握这两类规则就能正确使用 .github 文件夹。

2.在我看来

就是给你一个固定的模板，然后你可以在文件夹的最里层写一个你自己命名的文件（有固定后缀的）
